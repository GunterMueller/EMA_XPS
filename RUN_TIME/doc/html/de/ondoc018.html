RESTRIKTIONEN-EDITOR

Der Restriktionen-Editor
========================

Mit dem Restriktion-Editor können Restriktionen 
editiert und erstellt werden.
Zur Auswahl der betreffenden Wissensbasis ist das Feld
neben dem Label "Wissensbasis" anzuklicken. 
Das darunterliegenden Feld ermöglicht die Auswahl 
einer Restriktion. Soll eine neue Restriktion erstellt
werden, so ist der neue Name in das Feld Auswahl des 
Restriktion-Auswahlmenüs einzutragen.  

Das Bearbeiten-Menü:

- Diese Restriktion übernehmen...    
      Die geänderte Restriktion wird an die Wissensba-
      sis übergeben. 
- Diese Restriktion löschen...
      Diese Restriktion wird aus der Wissensbasis ent-
      fernt.
- Änderungen zurücknehmen... 
      Es wird wieder die unbearbeitete Restriktion ge-
      laden.
-Diese Restriktion erneut mit Constraints verknüpfen
      Diese Restriktion wird, entsprechend der im 
      Editor festgelegten Definitionen, erneut an die  
      zugehörigen Constraints und Constraintnetze
      gebunden.
-Alle Restriktionen erneut mit Constraints verknüpfen
      Alle in dieser Wissensbasis definierten Restrik-
      tionen werden an die zugehörigen Constraints und
      Constraintnetze gebunden. 

Eine noch nicht in die Wissensbasis übertragene Ände-
rung einer Restriktion wird durch ein Ausrufezeichen 
in der rechten oberen Ecke angezeigt. 

Durch Anwählen eines Wortes in den Textfeldern der
Editoren mit dem Textcursor und anschließendes Drücken
des [Hilfe]-Knopfes [F1] kann Hilfe zur Syntax der
Babylon- und LISP-Ausdrücke abgerufen werden.
